                                                                **CRM Database Project**
**Overview**
This project is a Customer Relationship Management (CRM) Database System designed to store, manage, and analyze customer-related data efficiently. The database supports core CRM operations such as customer management, lead tracking, sales activities, and reporting. The goal of this project is to demonstrate effective database design, normalization, and query optimization for real-world business use cases.

**Features**
Customer information management
Lead and contact tracking
Sales and interaction history
User and role management
Reporting and analytics support
Data integrity using constraints and relationships
Database Design
The database is designed using relational principles and follows normalization standards to reduce redundancy and improve data integrity.

**Key Entities**
Customers – Stores customer personal and contact details
Leads – Tracks potential customers and their status
Contacts – Manages individual contacts associated with customers
Users – CRM system users (sales reps, admins, managers)
Activities – Logs interactions such as calls, meetings, and emails
Sales/Deals – Tracks opportunities and deal progress

**Relationships**
One customer can have multiple contacts and activities
Leads can be converted into customers
Users can manage multiple customers and deals

**Technologies Used**
Database: MySQL 
Query Language: SQL
Tools: MySQL Workbench
Frontend: XAMP
